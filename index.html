Case Study 01 — BuildGuard
🔍 The Problem

The 2020 SolarWinds attack showed the world how devastating a compromised software build process could be. Attackers slipped malicious code into trusted builds and nobody noticed for months. I wanted to build something that would actually catch that.

🔨 What I Built

BuildGuard is a full-stack web tool that verifies the integrity of C software builds. It captures a cryptographic fingerprint of a clean build, then compares every subsequent build against it — flagging any tampering automatically and logging the full history.

⚙️ How It Works
1 Record a verified baseline build hash
2 Run verification on each new build
3 Compare hashes — flag mismatches instantly
4 Log and report all verification history

Case Study 02 — DarkShield
🔍 The Problem

Dark patterns are deceptive UX tricks used by websites to manipulate users — forcing subscriptions, hiding fees, and guilt-tripping people into decisions. My published research found 73% of Indian shoppers made poor purchase decisions because of them. I needed a tool to expose this.

🔨 What I Built

DarkShield is a browser-based detector. You enter a URL or paste raw HTML and it immediately scans for 6 categories of dark patterns using regex and keyword detection logic. It runs entirely client-side — no backend, no login, instant results.

🎭 Patterns Detected
🕐 Fake Urgency (countdown timers, scarcity claims)
☑️ Pre-ticked Boxes (opt-out by default)
💸 Hidden Fees (undisclosed charges)
😔 Confirm Shaming (guilt-trip opt-outs)
🔁 Forced Continuity (auto-renewals)
🎭 Disguised Ads (ads styled as content)
